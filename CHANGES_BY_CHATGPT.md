# Изменения в этой версии

Проверено аккуратно, без изменения дизайна и бизнес-логики сайта.

## Что изменено

1. `app/page.tsx`
   - Убран лишний `"use client"`.
   - Убран неиспользуемый `useState`.
   - Страница стала серверной оберткой для desktop/mobile блоков.

2. `components/MobileLanding.tsx`
   - Вынесены встроенные стили из JSX в `app/globals.css`.
   - Это уменьшает клиентский JavaScript, потому что большой CSS больше не лежит внутри клиентского компонента.
   - Hero-изображение оставлено через `next/image` в режиме `fill + priority`.
   - Второе крупное изображение автомобиля (`tmCarMainImg`) переведено на `next/image` в режиме `fill` без `priority`.
   - Иконки и изображения в мобильной форме/контактах переведены на `next/image` с явными размерами.

3. `app/globals.css`
   - Добавлен блок `MobileLanding extracted styles` — это те же стили, которые раньше были внутри `MobileLanding.tsx`.
   - Визуально сайт должен остаться прежним.

4. `tsconfig.json`
   - Добавлено `ignoreDeprecations: "6.0"`, чтобы `tsc` не падал из-за предупреждения TypeScript о старом `target: es5`.

## Проверка

В контейнере выполнена проверка TypeScript:

```bash
./node_modules/.bin/tsc --noEmit --pretty false
```

Ошибок TypeScript не обнаружено.

Полный `npm run build` в контейнере не удалось выполнить из-за отсутствия Linux SWC-бинарника Next.js в загруженном `node_modules` и запрета контейнера на скачивание пакетов. Это проблема среды проверки, а не кода проекта. На вашем компьютере или в Vercel сборка должна использовать корректные зависимости после `npm install`.

## Как безопасно применить

1. Распаковать архив.
2. Выполнить:

```bash
npm install
npm run build
```

3. Проверить сайт локально:

```bash
npm run dev
```

4. Если всё визуально совпадает — сделать commit и push.
